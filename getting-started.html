Getting Started

Set up your branded instance of {{brandName}}, your own style guide and component playground powered by the core library.

Installation

Get the package and scaffold your brand layer.

1

Install {{distName}}

Install the package as a dependency. This gives you the compiled component bundle in dist/ and the base brand scaffold in component-assets/.

bash 
npm install {{distName}}

You also need Sass available to compile your brand CSS. If you don't have it globally, add it as a dev dependency.

bash 
npm install --save-dev sass
2

Copy Your Brand Layer

Copy component-assets/ from the installed package into the root of your project. This is a one-time copy. It becomes your owned brand layer and is never touched by future {{distName}} updates.

bash 
cp -r node_modules/{{distName}}/component-assets ./component-assets

After copying, your project should include these directories:

directory 
your-project/
├── component-assets/        ← your brand layer (owned by you)
│   ├── scss/
│   │   ├── main.scss        ← your styles entry point (tokens + imports)
│   │   └── ...
│   ├── webfonts/            ← icon fonts
│   └── js/
├── components/              ← your component.json configs (you create these)
├── node_modules/
│   └── {{distName}}/
│       └── dist/            ← compiled JS bundle (from npm, read-only)
└── package.json

Configuration

Apply your brand identity to the SCSS and playground settings.

3

Configure Your Brand

Open component-assets/scss/main.scss. This is the single file that controls your visual identity. Set your Bootstrap variable overrides above the import and add any custom styles below it.

main.scss 
// Brand overrides - set these before the Bootstrap import
$primary:           #0072ce;
$secondary:         #6c757d;
$font-family-base:  'Inter', sans-serif;
$font-size-base:    0.875rem;

@import 'bootstrap/scss/bootstrap';

// Your custom styles below

Then update brand.json in your project root. This controls the playground's identity: the name shown in the sidebar and the built file names.

brand.json 
{
  "distName": "my-brand-ui",
  "globalName": "MyBrandUI",
  "title": "My Brand Design System",
  "version": "1.0.0",
  "hideHero": false,
  "hideWelcome": false
}

Build & Run

Compile your branded CSS and launch the playground.

4

Build Your CSS & Launch

Add these scripts to your package.json to compile your brand SCSS into CSS and start the playground server.

package.json 
"scripts": {
  "build:css": "sass --style compressed --quiet-deps component-assets/scss/main.scss dist/css/styles.min.css",
  "watch:css": "sass --watch --quiet-deps component-assets/scss/main.scss dist/css/styles.min.css",
  "start": "python3 -m http.server 8000"
}

Run the build, then start the server:

bash 
npm run build:css
npm start
# → http://localhost:8000

Component Playground

Document your components with an interactive props editor and live preview.

Building with AI

When generating component.json configs with an AI assistant, provide it with docs/COMPONENTS.md alongside your component's JS source. That file contains the full schema reference (prop types, template format, and dependency loading) so the output follows the correct patterns.

5

Add a Component Config

For each component you want in the playground, create a component.json file inside that component's directory. The playground reads this file to generate the live preview and interactive props panel.

component.json 
{
  "version": 1,
  "title": "Button",
  "description": "A primary action button.",
  "template": "<my-button btn-style=\"btn-primary\" label=\"Click Me\"></my-button>",
  "props": [
    {
      "id": "label",
      "label": "Label",
      "type": "text",
      "attribute": "label",
      "default": "Click Me"
    },
    {
      "id": "btn-style",
      "label": "Style",
      "type": "select",
      "attribute": "btn-style",
      "default": "btn-primary",
      "options": [
        { "label": "Primary", "value": "btn-primary" },
        { "label": "Secondary", "value": "btn-secondary" }
      ]
    }
  ]
}
Component dependencies

If a component depends on another to render (for example, a dropdown that uses a button internally), list those component paths in a dependencies array. The playground loads them before rendering the preview.

component.json 
{
  "version": 1,
  "title": "Dropdown",
  "dependencies": ["components/button/"],
  "template": "...",
  "props": [...]
}
6

Register in the Sidebar

Add your component to components/manifest.json. The playground reads this file to build the sidebar navigation. Without an entry here, the component won't appear.

manifest.json 
[
  { "name": "button", "path": "components/button/" },
  { "name": "modal",  "path": "components/modal/"  }
]

To group related components under a collapsible heading, use a children array:

manifest.json 
[
  {
    "name": "Form",
    "children": [
      { "name": "input",    "path": "components/input/"    },
      { "name": "checkbox", "path": "components/checkbox/" }
    ]
  },
  { "name": "button", "path": "components/button/" }
]

Using Components in Your App

Load the built library into any page or app.

7

Load the Library

Components self-register as custom elements when the bundle loads. Bootstrap CSS is bundled in automatically, so no separate stylesheet is required. Load your compiled brand CSS alongside it.

Via script tag (UMD)

Best for plain HTML pages. Drop the file in and components are immediately available.

html 
<link rel="stylesheet" href="dist/css/styles.min.css">
<script src="node_modules/{{distName}}/dist/{{distName}}.umd.js"></script>

<!-- Components are now available -->
<your-button variant="primary" label="Save"></your-button>
Via ES module

Best when bundling with Vite, Webpack, or another bundler.

javascript 
// All components self-register as a side effect
import '{{distName}}';

// Or import specific components by name
import { MyButton, MyModal } from '{{distName}}';
Enabling Dropdown and Tooltip support

Components that use Bootstrap JS (dropdowns, tooltips) check for window.bootstrap at connect time. If it isn't present yet, they wait for a bootstrap:ready event. Load Bootstrap JS and dispatch that event after it initializes:

html 
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"
  onload="document.dispatchEvent(new Event('bootstrap:ready'))">
</script>
<script src="node_modules/{{distName}}/dist/{{distName}}.umd.js"></script>

All other components are unaffected. Only include Bootstrap JS if your components use dropdowns or tooltips.

Staying Up to Date

Pull in core improvements without disrupting your brand or component configs.

8

Update the Core

Updating {{distName}} only changes the JS bundle in node_modules/. Your component-assets/, brand.json, and components/ configs are untouched.

bash 
npm update {{distName}}

When to review your SCSS

{{brandName}} follows semantic versioning. Use this as a guide for how much review each update needs:

Patch 1.0.x

Bug fixes only. No SCSS review needed, update freely.

Minor 1.x.0

New components or variants added. Check the changelog for any new SCSS variables.

Major x.0.0

Breaking changes. Diff your SCSS against the new base and review the changelog.

For major updates, diff the new base SCSS against your version to catch anything new:

bash 
diff -r node_modules/{{distName}}/component-assets/scss ./component-assets/scss
